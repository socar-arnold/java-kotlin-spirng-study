## 1. 실행 파이프라인 (TS와 대비)

```
[TS]   .ts   ──(tsc)──▶        .js   ──(node)──▶  실행
                ▲                        ▲
            컴파일러                  런타임/실행기

[JVM]  .java ──(javac)──▶  .class(바이트코드) ──(JVM)──▶  실행
.kt   ──(kotlinc)──▶
                ▲                                  ▲
         컴파일러(JDK 안)                       실행기
```

- `javac`/`kotlinc` = `tsc` 자리 (컴파일러)
- `JVM` = `node`(V8) 자리 (실행기)
- **흔한 오해 교정:** `.java → .class` 변환은 **JVM이 아니라 컴파일러(javac)**가 한다. JVM은 *그다음* `.class`를 실행한다.

### 바이트코드(.class)
- OS·CPU에 독립적인 중간 표현 → **같은 .class가 어디서든 JVM만 있으면 동일 동작** (Write Once, Run Anywhere).
- 사람이 읽는 텍스트가 아니라 바이너리. (`javap -c App.class`로 역어셈블해 구경은 가능 — Phase 2에서)

## 2. JDK / JRE / JVM (포함관계)

```
JDK (개발 키트)  ⊃  JRE (실행 환경)  ⊃  JVM (가상 머신/실행기)
```

| 약어 | 풀네임 | 역할 | TS/Node 비유 |
|---|---|---|---|
| **JVM** | Java **Virtual Machine** | 바이트코드를 실제 실행하는 엔진 | V8 엔진 |
| **JRE** | Java **Runtime Environment** | JVM + 표준 라이브러리 (실행 전용) | node 런타임 |
| **JDK** | Java **Development Kit** | JRE + 컴파일러(javac)·도구 (개발+실행) | node + tsc + 개발툴 |

- **컴파일하려면 → JDK** 필요 (javac 포함)
- **실행만 하려면 → JRE/JVM** 면 충분
- 실무: Docker 배포 이미지엔 JDK 대신 **JRE만** 넣어 용량 절감 (Phase 6에서 활용)

### 기억 트릭
- **V**irtual machine → 굴리는 엔진 (제일 안쪽)
- **R**untime → 실행(run)만 (중간)
- **D**evelopment → 개발(만들기)까지 (제일 바깥, 풀세트)

## 3. 컴파일 vs 파싱 (헷갈림 정리)

**파싱은 컴파일의 대등한 짝이 아니라, 컴파일 "안의 한 단계"다.**

```
소스 텍스트 → ① 렉싱(쪼개기) → ② 파싱(구조 만들기) → ③ 의미분석 → ④ 코드 생성
"val x = 1"   [val][x][=][1]      트리(AST)            타입체크 등    .class/.js/기계어
```

- **렉싱(lexing/토크나이징):** 텍스트를 토큰으로 **쪼갠다**. (← "쪼갠다"는 사실 이것)
- **파싱(parsing):** 쪼갠 토큰을 **문법 트리(AST)로 조립**해 "말이 되는 구조인지" 세운다.
  - 문장 비유: "나는 밥을 먹었다" → 쪼개기는 `[나는][밥을][먹었다]`, 파싱은 `주어+목적어+동사` 트리로 구조화.
  - 구조가 깨지면 여기서 **Syntax Error**.
- TS에서 매일 겪는 것: `tsc`/Babel/ESLint/Prettier 모두 내부에서 코드를 파싱해 **AST**를 만든다. 빨간 줄 `Syntax Error` = 파싱 실패.

### 정의 교정
| 처음 이해 | 교정 |
|---|---|
| 컴파일 = 고차원 언어 → 저차원(바이너리) | 거의 맞음. 단 결과가 꼭 바이너리는 아님(tsc→JS는 텍스트). "더 낮은/다른 표현으로 **번역**"이 정확 |
| 파싱 = 언어를 음절단위로 쪼갠다 | "쪼개기"는 **렉싱**. 파싱은 토큰을 **문법 트리로 조립·구조화**하는 단계 |

- **컴파일은 javac/kotlinc(JDK)가, 실행은 JVM(⊂JRE)이 한다.**
