# 학습 트랙 설계 — AI 튜터 주도 백엔드 양성

> 작성일: 2026-05-22
> 목표: TypeScript에 익숙하지만 JVM은 처음인 학습자를, AI 튜터 주도 인터랙티브 세션으로
> 미드~시니어 Kotlin 백엔드 개발자 수준까지 끌어올린다.

---

## 1. 배경 & 동기

기존 [CURRICULUM.md](../../../CURRICULUM.md)는 Kotlin 중심의 "미드~시니어" 자체 완결형 교육 콘텐츠(Phase 0~12)다.
학습자는 시중 책/강의(자바의 신, 김영한 스프링·JPA, 박우빈 테스팅 등)를 참고 자료로 고려했으나,
**해당 자료를 소유하고 있지 않다.** 따라서 책을 가리키는 대신,
**AI 튜터가 같은 주제를 원본 교육 자료로 직접 집필하고, 인터랙티브 세션으로 학습을 진행**한다.
(책 본문 복제가 아니라 동일 주제를 원본으로 재집필.)

---

## 2. 대상 페르소나

**"TypeScript에 익숙하지만 JVM은 처음인 개발자"**

- **이미 아는 것:** 타입, async/await(Promise), OOP·함수형 기초, 모듈/패키지 개념
- **새로운 것:** JVM·바이트코드·GC, 정적 타입 디테일, 스레드 모델(JS 이벤트 루프와 다름),
  Kotlin/Java 문법, Spring DI, JPA/ORM

**설계 적용 원칙:**
- 모든 개념을 TS에 매핑해서 가르친다 (새로 배우기보다 "아는 것과 연결").
  - TS `Promise`/`async-await` → Kotlin 코루틴
  - TS 구조적 타이핑 → Kotlin 명목적 타이핑
  - TS 유니온 타입 → Kotlin sealed class
  - NestJS DI/데코레이터 → Spring DI/애너테이션
  - TypeORM·Prisma → JPA/Hibernate
  - `null`/`undefined` → Kotlin 널 안전성(`?.`, `?:`)
- Java 온램프는 얕게 — 프로그래밍 자체는 아니까 "변수란?" 같은 건 생략.
- 페이스 강약: 프로그래밍 공통 개념은 빠르게, 진짜 JVM-새로운 것(스레드/GC/바이트코드/Spring 매직)은 천천히.

## 5. Stage 순서 (가볍게 페이스, 누적 주차)

| Stage | 내용 | 자료 출처 | 누적 |
|---|---|---|---|
| 0. Java 온램프 | JVM·생태계·문법·OOP 개념만 (남 코드 읽기용, TS 매핑) | **신규 집필** | ~4주 |
| 1. Kotlin 기초 | Phase 1 + 비기너 쿠션 + OOP 사고(역할·책임·협력) | Phase 1 + 보강 | ~18주 |
| 2. 빌드·JVM·동시성 | Phase 0 / 2 / 3 + 미션 (코루틴 = TS async 대비) | 기존 | ~30주 |
| 3. 백엔드 | Phase 4 (Spring·JPA·테스트) + 미션(블랙잭/로또) | 기존 + 깊이 보강 | ~46주 |
| 4. 심화 | Phase 5~12 (DB·운영·아키텍처·성능·MQ·보안·장애) | 기존 | 이후 |
| 5. 분산·시스템디자인 | **Phase 13 신설** (§7 참조) | 신규 집필 | 이후 |
| 병렬 | CS — 면접·실전 빈출 우선, 매주 1개 | 신규 큐레이션 | 전 구간 |

**트레이드오프 (기록):** 비기너에게 동시성(Phase 3)이 Spring보다 먼저 오는 건 무거울 수 있다.
기본은 기존 순서를 따르되, 학습 중 부담되면 Stage 2의 동시성을 Spring 뒤로 미루는 변형을 허용한다.

### CS 병렬 트랙 (면접·실전 빈출)
- 네트워크: TCP/UDP, HTTP/HTTPS·TLS, HTTP 1.1/2/3, 쿠키·세션·토큰, CORS
- 운영체제: 프로세스 vs 스레드, 컨텍스트 스위칭, 가상메모리·페이징, 동기화·데드락
- 자료구조/알고리즘: 해시·트리·정렬, 시간복잡도 (코딩테스트 기초)
- DB CS: 인덱스 원리(B-tree), 트랜잭션·ACID, 격리수준, 정규화
- (선택) 디자인패턴 기초
- 역할 분담: CS 트랙은 "면접에서 말로 설명할 수준"의 개념을 먼저, 커리큘럼 Phase는 "코드로 구현"을 나중에 → 예습→실습 연계. 매주 주제는 그 주 메인 학습과 연관된 것을 우선 배치.

## 6. Java 온램프 범위 (신규 집필, 얕게)

JVM/JDK가 뭔지 → 변수·타입·제어문(빠르게, TS와 대비) → 클래스·객체·상속·인터페이스 →
컬렉션/제네릭 맛보기 → Java만의 관용구(체크 예외, getter/setter, 제네릭 소거) →
"Kotlin은 이걸 왜 바꿨나" 비교. **목표는 작성이 아니라 읽기** — 깊이 안 판다.

## 7. 미드~시니어 갭 보강 (확정)

기존 커리큘럼은 이미 페이지네이션(offset+cursor), HikariCP 튜닝, 분산 락(캐시 스탬피드),
읽기 복제본(CQRS), Kafka 파티셔닝, 멱등성, OWASP, 회복탄력성까지 커버해 기술 폭은 미드 레벨을 넘는다.
다만 시니어로 확실히 올라서기 위해 아래를 보강한다.

### 7-1. Phase 13 신설 — 분산 시스템 & 시스템 디자인
- **분산 트레이싱·관측성 심화:** OpenTelemetry/Zipkin 분산 추적, SLI/SLO/알람 설계 (기존 Phase 6 로깅/메트릭 위)
- **분산 트랜잭션 Saga:** 보상 트랜잭션, 분산 일관성 (기존 Phase 10 Outbox 위)
- **API 설계·계약 심화:** API 버저닝, OpenAPI 자동 문서화, 계약 테스트(consumer-driven)
- **시스템 디자인:** 대용량 트래픽 설계, 로드밸런싱, rate limiting, CDN, 확장성 사고 (시니어 면접·실무 핵심)

### 7-2. Phase 2/8 보강 — JVM 프로파일링 실전
- async-profiler/JFR, 힙덤프 분석, GC 튜닝 실전 진단 (기존 GC 이론·부하테스트 위)

### 7-3. Phase 4-F 보강 — OAuth2/OIDC 전체 플로우
- 인가 서버·소셜 로그인·토큰 플로우 전체 (기존 JWT 위)

### 7-4. 시니어 소프트스킬 — 세션 프로토콜·메모리에 상시 반영
- 코드 리뷰, 기술 문서 작성, 레거시 코드 다루기, 설계 의사결정
- 별도 Phase가 아니라 매 세션 리뷰를 "시니어 코드리뷰처럼" 하고, 설계 트레이드오프를 항상 말로 설명하게 함

### 7-5. 미보강 항목 (의도적 보류 / 향후 후보)
- JSP 2.3: 레거시 → 후순위·선택. 서블릿/웹 동작 원리는 Phase 4에서 간접 흡수.
- 이벤트 소싱: CQRS(Phase 7)·Kafka(Phase 10)로 인접 커버, 단독 주제는 보류.
- 쿠버네티스 심화·IaC: Phase 6/12에서 Docker·kubectl 롤백 수준만. 클라우드 네이티브 심화는 향후 후보.

## 8. 세션 프로토콜 (1세션 = 1주제)

1. 지난 내용 1분 복습 체크
2. **WHY** — 나쁜 코드/문제 상황 먼저 (코드부터 보여주지 않는다)
3. **WHAT** — 개념 + trade-off (왜 이 선택 / 언제 유리 / 언제 과한가) + TS 매핑
4. **HOW** — 문제·미션 제시 → 학습자가 **직접 타이핑(복붙 금지)** → TDD Red→Green→Refactor →
   튜터가 **시니어 코드리뷰처럼** 리뷰·피드백
5. 마무리 요약 + 다음 예고 → `PROGRESS.md` 갱신

기존 교육 원칙(WHY→WHAT→HOW, Trade-off 필수, TDD 철저, Idiomatic Kotlin, YAGNI)을 그대로 따른다.
